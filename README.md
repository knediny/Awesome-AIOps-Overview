## Datasets
- [CUHK] [Loghub](https://github.com/logpai/loghub) - 香港中文大学提供的不同系统和应用程序的日志数据集合。
- [CUHK] [LogPub](https://github.com/logpai/LogPub) - 香港中文大学发布的一个包含多个来源公开日志数据的库。
- [CUHK] [bughub](https://github.com/logpai/bughub) - 由香港中文大学提供，包含各种软件项目的漏洞报告集合。
- [NAB] [The Numenta Anomaly Benchmark(NAB)](https://github.com/numenta/NAB) - Numenta公司制定的异常检测基准数据集。
- [Microsoft Azure] [Azure Public Dataset](https://github.com/Azure/AzurePublicDataset) - 微软云服务平台Azure提供的公共数据集。
- [Google] [Cluster Traces](https://github.com/google/cluster-data) - 谷歌发布的集群追踪数据，反映了其生产环境中的工作负载情况。
- [Backblaze] [Hard Drive Dataset](https://www.backblaze.com/b2/hard-drive-test-data.html) - Backblaze公司提供的硬盘驱动器测试数据集。
- [Baidu] [SMART Dataset of PAKDD CUP 2020](https://pan.baidu.com/share/link?shareid=189977&uk=4278294944#list/path=%2FS.M.A.R.T.dataset) - 百度为PAKDD CUP 2020竞赛提供的智能硬盘监控属性数据集。
- [Alibaba] [SSD SMART logs and failure data](https://github.com/alibaba-edu/dcbrain/tree/master/ssd_open_data) - 阿里巴巴提供的SSD固态硬盘SMART日志与故障数据。
- [Alibaba] [Alibaba Cluster Trace Program](https://github.com/alibaba/clusterdata) - 阿里巴巴的集群跟踪数据程序，记录了集群运行状况。
- [Tsinghua] [AIOps Challenge Dataset](https://competition.aiops-challenge.com/home/competition/) - 清华大学提供的针对人工智能运维挑战的数据集。
- [Yahoo] [A Labeled Anomaly Detection Dataset](https://webscope.sandbox.yahoo.com/catalog.php?datatype=s&did=70) - 雅虎提供的带标签的异常检测数据集。
- [Mike Sconzo] [Security Repo](https://www.secrepo.com/) - Mike Sconzo创建的安全相关的数据仓库。
- [AIT] [AIT](https://zenodo.org/records/5789064#.YkFnZWJBxhE) -- [AIT aecid](https://github.com/ait-aecid/kyoushi-environment) - 奥地利技术研究所提供的信息技术安全训练和模拟环境数据集。


## benchmarks
- [arXiv '22] [Constructing Large-Scale Real-World Benchmark Datasets for AIOps](https://arxiv.org/abs/2208.03938) - 讨论构建大规模真实世界AIOps基准数据集的方法和挑战。
- [ASPLOS '19] [An Open-Source Benchmark Suite for Microservices and Their Hardware-Software Implications for Cloud and Edge Systems](https://dl.acm.org/doi/10.1145/3297858.3304013) - 2019年在ASPLOS会议上提出的微服务基准测试套件，探讨了微服务对云计算和边缘计算系统硬件与软件的影响。
- [Google Cloud] [Online Boutique (A microservices demo application)](https://github.com/GoogleCloudPlatform/microservices-demo) - 谷歌云平台提供的微服务演示应用程序"Online Boutique"，用于展示云原生应用开发和管理的最佳实践。
- [Fudan] [Train Ticket (A benchmark microservice system)](https://github.com/FudanSELab/train-ticket) - 复旦大学的研究团队开发的一个名为"Train Ticket"的微服务基准系统，用于研究微服务架构的各类问题。
- [Weaveworks] [Sock Shop (A microservices demo application)](https://microservices-demo.github.io/) - Weaveworks公司提供的一个叫"Sock Shop"的微服务演示应用程序，旨在展现如何构建和部署微服务架构应用。


## Research Frontiers
- [Automatic Root Cause Analysis via Large Language Models for Cloud Incidents](https://arxiv.org/abs/2305.15778) - 介绍如何利用大型语言模型自动分析云服务事件的根本原因。
- [LLM As DBA](https://arxiv.org/abs/2308.05481) -- [project](https://github.com/TsinghuaDatabaseGroup/DB-GPT) - 探索将大型语言模型作为数据库管理员（DBA）使用，以辅助数据库相关任务，提高效率和性能。
- [Recommending Root-Cause and Mitigation Steps for Cloud Incidents using Large Language Models](https://arxiv.org/abs/2301.03797) - 使用大型语言模型推荐云服务事故的根本原因及缓解措施。
- [A Holistic View of AI-driven Network Incident Management](https://www.microsoft.com/en-us/research/uploads/prod/2023/09/LLM4IcMs___HotNets__23-6.pdf) - 微软研究院提出的AI驱动的网络事件管理全景视角，描述了AI如何改善事件管理流程。
- [Assess and Summarize: Improve Outage Understanding with Large Language Models](https://arxiv.org/abs/2305.18084)- 利用大型语言模型评估和总结信息，以改进对停电(中断)事件的理解。
- [PACE-LM: Prompting and Augmentation for Calibrated Confidence Estimation with GPT-4 in Cloud Incident Root Cause Analysis](https://arxiv.org/abs/2309.05833) - 针对GPT-4在云服务事故根因分析中的置信度校准，提出了提示和增强技术。
- [Log Parsing: How Far Can ChatGPT Go?](https://arxiv.org/abs/2306.01590) -- [project](https://github.com/LogIntelligence/log-analytics-chatgpt) - 探讨ChatGPT在日志解析方面的能力和局限性。
- [Log-based Anomaly Detection based on EVT Theory with feedback](https://arxiv.org/abs/2306.05032) - 结合极值理论（EVT）和反馈机制，提出基于日志的异常检测方法。
- [Exploring the Effectiveness of LLMs in Automated Logging Generation: An Empirical Study](https://arxiv.org/abs/2307.05950) -- [project](https://github.com/LogStudySE/LogStudy) - 实证研究大型语言模型在自动生成日志方面的有效性。
- [Prompting for Automatic Log Template Extraction](https://arxiv.org/abs/2307.09950) - 研究如何使用提示技术来自动提取日志模板。
- [LogPrompt: Prompt Engineering Towards Zero-Shot and Interpretable Log Analysis](https://arxiv.org/abs/2308.07610) - 提出一种提示工程方法，旨在实现零样本学习和可解释的日志分析。
- [LogGPT: Exploring ChatGPT for Log-Based Anomaly Detection](https://arxiv.org/abs/2309.01189) - 探索使用ChatGPT进行基于日志的异常检测的可能性。
- [LLMParser: A LLM-based Log Parsing Framework](https://arxiv.org/abs/2310.01796)  - 提出一个基于大型语言模型的日志解析框架。
- [Empower Large Language Model to Perform Better on Industrial Domain-Specific Question Answering](https://arxiv.org/abs/2305.11541) -- [project](https://github.com/ModelInteraction/MSQA) - 加强大型语言模型在工业领域特定问答任务上的表现。
- [OWL: A Large Language Model for IT Operations](https://arxiv.org/abs/2309.09298) -- [project](https://github.com/HC-Guo/Owl) - OWL是一个专门为IT运维任务设计的大型语言模型。
- [An Empirical Study of NetOps Capability of Pre-Trained Large Language Models](https://arxiv.org/abs/2309.05557) -- [project](https://huggingface.co/datasets/NASP/neteval-exam) - 对预训练的大型语言模型进行网络运营能力的实证研究。
- [OpsEval: A Comprehensive Task-Oriented AIOps Benchmark for Large Language Models](https://arxiv.org/abs/2310.07637)] -- [project](https://opseval.cstcloud.cn/content/home#home) - OpsEval是针对大型语言模型在自动化IT运维方面的综合任务导向基准测试。
- [codefuse-ai/codefuse-devops-eval](https://github.com/codefuse-ai/codefuse-devops-eval) -- paper comming soon?- CodeFuse AI发布的面向开发运维的评估工具，相关论文即将发布。

- [ISSRE '16] [Experience Report: System Log Analysis for Anomaly Detection](https://ieeexplore.ieee.org/abstract/document/7774521) - 分享了使用系统日志分析进行异常检测的经验报告。
